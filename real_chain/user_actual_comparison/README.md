# 用户提供的真实 UI 对比图

本目录保存用户提供的三张真实对比图，用于展示 `image2-ui-code-restorer` 与普通 Codex 还原之间的差异。

## 图片说明

| 文件 | 角色 | 尺寸 |
|---|---|---:|
| `assets/01_no_skill_restore.png` | 未使用技能还原 | `1536 x 1024` |
| `assets/02_original_prototype.png` | 原型图 / Image2 生成图 | `2402 x 1484` |
| `assets/03_with_skill_restore.png` | 使用技能还原 | `2940 x 1602` |

## 核心观察

普通还原版本完整度不低，但页面结构发生了较明显的再设计：新增顶部导航、咨询记录条、专家卡片数据与原型图差异较大，右侧统计模块也重新组织过。

技能还原版本更接近原型图的信息架构：保留了标题区、筛选区、热门领域、专家卡片网格、右侧统计卡、领域分布、专家优势和使用指南等核心模块。

## 使用建议

这组三图适合用作“为什么需要专门的截图还原技能”的静态对比案例。建议表达重点是：

`普通方式可以做出页面，但更像重新设计；技能方式更像按原图拆结构再还原。`

不建议宣传成像素级 1:1，因为当前技能版仍存在视口比例、数据文本、字号密度和局部细节差异。
