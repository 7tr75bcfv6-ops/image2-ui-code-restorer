## 一、准备工作

1. 准备一张你要还原的 UI 目标图（来自 Image2）
2. 统一分辨率：建议 `1920×1080` 或与你目标页面比例一致
3. 保持图中内容可辨：避免过多噪点、压缩马赛克
4. 先给出“起点截图”和“终点页面”的参考（对比时必备）

## 二、安装与环境

### 2.1 最小环境

- 浏览器支持：Chrome/Edge（推荐最新版）
- 截图能力：Screen Studio（用于录屏）
- 视频工具：HyperFrames by HeyGen（用于剪辑）

### 2.2 插件安装（按你真实入口执行）

- Step 1：下载/获取 `image2-ui-code-restorer`

- Step 3：在“权限/目录”中允许读写你的项目目录
- Step 4：重启 Codex 或刷新插件列表确认可见

## 三、使用流程（标准版）

### 3.1 快速流程

1. 新建工程文件夹：`demo-image2-ui`
2. 将 Image2 输出图放入 `inputs/`
3. 启动插件并选择该图片
4. 选择输出目标（HTML / React / Vue）
5. 选择参数后点击 `开始还原`
6. 下载并预览（建议先跑本地静态页）
7. 对出现偏差的区域进行手工二次修复

### 3.2 典型参数（演示推荐）

| 参数 | 默认值 | 推荐值 | 用途 |
| --- | --- | --- | --- |
| 识别模式 | `balanced` | `fast`（演示） / `strict`（最终） | 控制速度与准确率权衡 |
| 输出框架 | `html` | 你发布场景常用框架 | 保持下游集成一致 |
| 样式策略 | `semantic` | `pixel-accurate`（关键页） | 控制可还原程度 |
| 组件颗粒度 | `medium` | `fine`（复杂布局） | 提高可编辑性与后续维护性 |
| 代码压缩 | `off` | `off`（演示） | 便于看清结构 |
| 自动修复 | `on` | `on` | 处理简单边缘问题 |

> 你可以把“样式策略”从 `fast` 切到 `strict`，作为“无插件对比”中间加速段的对照。

## 四、对比素材拍摄（无插件 vs 插件）

### 4.1 无插件组（对比段）

- 用同一张图片起步
- 手动搭建到一个可见阶段（头尾两个关键点）
- 中间用加速片段（避免展示大量重复拖拽操作）

### 4.2 插件组（核心组）

- 从同一张起始图开始
- 插件导入 → 一键开始 → 结果产出 → 预览运行
- 结果截图必须和无插件组同分辨率

### 4.3 时间点建议

- 插件组从 36s 开始，放置在短视频的视觉高潮
- 无插件组放 49s 前后，体现“传统路径”成本

## 五、常见失败案例与排查

- 字体偏差：补齐字体加载方式并统一字号比例
- 间距失真：检查像素单位是否被缩放（尤其 2K/4K 录屏来源）
- 按钮状态缺失：补手工交互状态（hover/focus/disabled）
- 颜色偏差：确认图像是否经平台压缩导致色温偏移
- 版心错位：统一画布尺寸（建议固定为 1920x1080）

排查顺序：

1. 对比输入图与输出图的尺寸是否一致
2. 核对“输出框架”与项目一致（Vue/React）
3. 清理缓存后重新还原一次
4. 参数切到 `strict` 重跑
5. 仅对偏差区域进行手工修复
