# 真实 UI 对比报告：普通还原 vs image2-ui-code-restorer

## 结论先行

这组三图已经足够支撑首发图文案例。最适合的表达不是“技能版更漂亮”，而是：

`普通还原更像重新设计，技能还原更像按原型结构复原。`

## 普通还原版本观察

优点：

- 页面完整，能形成一个可用后台界面。
- 顶部导航、用户区、卡片列表、右侧统计区都有实现。
- 视觉清爽，按钮、标签和卡片样式统一。

主要偏差：

- 新增了原型图没有强调的顶部导航和咨询记录条，整体信息架构被改写。
- 专家卡片的人物头像变成首字母圆形头像，偏离原型中的真实专家头像。
- 专家卡片字段和数据变化较大，例如价格、服务单、评分、专家描述均与原型不一致。
- 右侧模块的统计呈现、领域分布、专家优势与原型图的视觉节奏不同。
- 页面更像“根据主题重新做了一版管理页”，不是按截图逐模块还原。

## 技能还原版本观察

优点：

- 核心页面骨架更接近原型图：标题区、筛选区、热门领域、专家卡片网格、右侧统计卡、领域分布、专家优势和使用指南都保留。
- 专家卡片保留真实头像，而不是替换成抽象字母头像。
- 右侧栏模块顺序和原型图更接近。
- 色彩体系、浅色背景、蓝绿橙状态色和卡片边框更贴近原型。
- 更符合 `image2-ui-code-restorer` 的工作流价值：先识别模块，再尽量按原图结构生成。

仍然存在的偏差：

- 三张图视口尺寸不一致，不适合直接做严格像素级评分。
- 技能版宽高比为 `1.835`，原型图为 `1.619`，视觉横向展开更多。
- 字号和留白略偏大，整体密度比原型更松。
- 部分数据语义没有完全保留，例如评分从 `98% / 97%` 变成 `4.8%`，服务单变成 `0 单`。
- 局部交互状态不同，例如右下角出现“专家库模块已打开”提示。
- 底部分页/条数区域与原型图呈现不完全一致。

## 是否还需要补充素材

如果只做静态图文案例，目前已经够用。

如果要做更严谨的 GitHub 展示，建议再补：

- 同一视口尺寸下导出的三张图，最好统一为 `1920 x 1080` 或 `1440 x 900`。
- 两个还原版本的运行页面地址或本地 HTML 文件。
- 技能调用过程的简短日志，证明确实调用了 `image2-ui-code-restorer`。
- 是否允许公开使用 `铂宸云慧` 这个品牌名和页面内容。

如果不想补也可以发布，但文案里应避免写“像素级 1:1”，建议写“结构还原更接近原型”。
